## 🏫 정리한 내용

ETag는 Entity Tag의 약자입니다.

ETag는 특정 리소스의 식별 코드를 의미하며 보통 2가지 용도로 주로 사용됩니다.
1. 리소스 캐싱 : 리소스의 변화가 없으면 클라이언트 측에 캐싱되어있는 데이터를 사용하라고 알려주는 용도다.
2. 충돌 방지 : 리소스 변화 요청을 수행하기 이전에 클라이언트가 갖고 있는 데이터가 현재 데이터와 다르면 변화 요청을 수행하지 않는다.
